# BigQuery Release Hub 🚀

A real-time developer feed and interactive social composer for Google Cloud BigQuery updates. This application pulls official BigQuery release notes from the RSS feed, parses the contents into granular category updates, caches results to improve performance, and provides a sleek dashboard to filter, search, and draft tweets to share on X (Twitter).

---

## 🌟 Key Features

*   **Live RSS Fetching & Parsing**: Automatically downloads and parses the latest release updates from the Google Cloud BigQuery Atom feed.
*   **Performance Cache (TTL)**: Stores processed records in a local `feed_cache.json` with a 15-minute Time-To-Live (TTL) to avoid redundant network requests.
*   **Granular Category Splitting**: Groups update text blocks automatically under specific categories: `Feature`, `Announcement`, `Issue`, `Changed`, `Deprecated`, and `General`.
*   **Interactive Search & Filters**: Search notes instantly by keyword or filter them by clicking category badges or stats cards.
*   **X (Twitter) Composer Workspace**: 
    *   One-click "Share on X" to open a live composer with pre-filled content.
    *   Pre-calculated hashtag additions and URL references.
    *   Dynamic character count status with a custom SVG progress ring.
    *   Warns or locks publishing if the draft exceeds Twitter's 280-character limit.
    *   Supports quick clipboard copying or direct browser intent redirection.

---

## 🛠️ Project Structure

*   [`app.py`](app.py): Core Flask application serving HTML templates and api data endpoints, handling XML Parsing, caching, and document classification.
*   [`templates/index.html`](templates/index.html): HTML5 single-page dashboard containing theme glow effects, list structures, and composer modal overlay.
*   [`static/js/app.js`](static/js/app.js): Controls global client states, fetches updates, coordinates DOM renders, and handles composer interaction logic.
*   [`static/css/style.css`](static/css/style.css): Modern CSS stylesheet using CSS variables, custom scrollbars, toast notifications, responsive flex/grid wrappers, and glassmorphic card styles.
*   [`feed_cache.json`](feed_cache.json): Autogenerated local cache file storing structured JSON data of the release notes feed.

## ⚙️ Installation & Setup

### Prerequisites
Make sure you have **Python 3.x** and **pip** installed on your system.

### 1. Install Dependencies
Run the following command to install required Python libraries:
```bash
pip install flask requests beautifulsoup4
```

### 2. Run the Application
Start the Flask development server:
```bash
python app.py
```
By default, the server will launch on port `5000`.

### 3. Open in Browser
Visit the following address in your browser:
```
http://127.0.0.1:5000/
```

## ⚡ API Endpoint Reference

### `GET /api/release-notes`
Fetches and returns the array of structured release notes.

*   **Query Parameters**:
    *   `refresh` (optional): Set to `true` to force bypass the local cache validation and fetch live data directly from the Google Cloud feed. (e.g., `/api/release-notes?refresh=true`).
*   **Success Response Schema**:
    ```json
    {
      "status": "success",
      "last_updated": 1781864043,
      "count": 1,
      "data": [
        {
          "date": "June 17, 2026",
          "updated_raw": "2026-06-17T12:00:00Z",
          "link": "https://cloud.google.com/bigquery/docs/release-notes",
          "updates": [
            {
              "category": "Feature",
              "html": "<p>BigQuery now supports new SQL analytical features...</p>",
              "text": "BigQuery now supports new SQL analytical features..."
            }
          ]
        }
      ]
    }
    ```
